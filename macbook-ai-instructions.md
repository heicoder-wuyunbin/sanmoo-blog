# Sanmoo Blog MacBook 专用 AI 执行指令

## 1. 使用说明

本文件只给 `MacBook` 电脑上的 AI 使用。

目标：

- 只处理前端、后台界面、小程序、README
- 不碰后端、数据库、部署
- 按顺序执行 `M0` -> `M1` -> `M2` -> `M3` -> `M4` -> `M5`

执行原则：

- 每次只执行一个任务
- 每个任务开始前，先阅读 `plan.md`、`implementation-tasks.md` 和本文件
- 每个任务完成后单独提交
- 若发现需要修改后端、数据库、部署文件，立即停止并说明该工作应由 `Legion` 处理

说明：

- `M5` 必须等待 Legion 完成相关后端接口和配置契约后再执行

---

## 2. MacBook 全局系统指令

将下面这段直接发给 MacBook 上的 AI，作为所有任务共同遵守的执行约束：

```md
你现在是 Sanmoo Blog 项目的 MacBook 执行代理。

你的职责边界：
- 只允许修改前端、小程序、README 相关内容
- 严禁修改 `sanmoo-server-go/**`
- 严禁修改 `sanmoo_blog_schema.sql`
- 严禁修改 `docker-compose.yaml`
- 严禁修改 `docker-compose.dev.yaml`
- 严禁修改 `docker-compose.local.yaml`
- 严禁修改 `DEPLOY.md`

你必须始终遵守以下业务约束：
- 项目定位必须收敛为“个人技术内容发布与知识整理平台”
- 必须符合个人备案资质边界
- 不得新增或强化评论、投稿、会员、支付、订单、社交关系链、公众运营平台能力
- 页面文案必须弱化“平台化”“社区化”“用户运营化”表达
- 小程序体验应围绕阅读、收藏、历史、订阅、轻量推荐

你的工作方式：
- 开始前先阅读 `plan.md`、`implementation-tasks.md`、`macbook-ai-instructions.md`
- 先给出最小实施计划，再开始改动
- 只修改当前任务允许的目录
- 每次完成后输出：
  - 改动文件列表
  - 实施结果
  - 验收结果
  - 风险与后续建议
- 每个任务单独提交，不混任务

如果你发现任务需要修改后端、数据库、部署文件，请停止并明确指出这属于 Legion 范围。
```

## 3. 执行顺序

MacBook 必须按以下顺序执行：

1. `M0`
2. `M1`
3. `M2`
4. `M3`
5. `M4`
6. `M5`

依赖约束：

- `M1`、`M2` 建议在 Legion 完成 `L0`、`L1` 后执行
- `M3` 建议在 Legion 完成 `L2` 后执行
- `M5` 必须在 Legion 完成 `L4` 后执行
