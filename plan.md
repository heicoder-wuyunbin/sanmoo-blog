### 1.2 核心问题诊断

**最大痛点：SEO 能力弱（SPA 架构导致）**
- 文章详情页 meta 标签是静态的，搜索引擎抓取不到动态内容
- 没有结构化数据（JSON-LD），搜索结果展示效果差
- 没有 canonical 标签，可能产生重复内容问题
- 面包屑导航缺失，不利于搜索引擎理解站点结构

**次要痛点：用户互动手段单一**
- 无点赞/喜欢功能（个人备案可以做，纯计数）
- 无分享功能（可以做，跳转第三方平台）
- 无随机文章功能（增加停留时长）

**体验优化点：**
- 代码块无一键复制（技术博客必备）
- 无阅读时长估算
- 无返回顶部按钮
- 搜索结果无高亮

## 二、个人备案合规边界（红线）

> **重要：所有功能必须在个人备案合规范围内设计**

### 2.1 禁止功能（绝对不能做）

| 功能 | 原因 | 替代方案 |
|------|------|----------|
| 用户评论/留言 | 交互式内容需企业资质 | 无替代，保持纯展示 |
| 用户注册/发帖 | UGC 内容需企业资质 | 仅保留管理员账号 |
| 论坛/社区 | 交互式社区需专项备案 | 无替代 |
| 付费内容/知识付费 | 经营性活动需企业资质 | 内容全部免费 |
| 广告联盟/商业广告 | 经营性活动需企业资质 | 可放个人收款码（打赏性质） |
| 邮件订阅营销 | 可能被认定为营销行为 | 用 RSS 订阅替代 |

### 2.2 允许功能（安全区）

- ✅ 文章展示/阅读
- ✅ 点赞/喜欢（纯计数，无用户发布内容）
- ✅ 分享到第三方平台（跳转链接形式）
- ✅ 搜索功能
- ✅ RSS 订阅
- ✅ 本地收藏/历史记录
- ✅ 打赏/捐赠（个人性质，非强制）
- ✅ 友情链接（个人博客互换，非商业）
- ✅ 关于/联系页面（个人介绍）

## 四、重点功能详细设计

### 4.1 SEO 优化方案（Phase 0 核心）

#### 方案选择：轻量 SEO 增强（推荐）

**不建议**：马上重构为 SSR（成本太高，ROI 低）
**建议**：先用 react-helmet-async + 后端预渲染关键页面

**具体措施：**

1. **动态 meta 标签**
   - 首页：title + description + og tags（从 settings 读取）
   - 文章详情：title（文章标题）+ description（文章摘要）+ og:image（封面图）
   - 分类/标签/专题页：对应名称 + 描述

2. **结构化数据（JSON-LD）**
   - BlogPosting 类型（文章详情页）
   - BreadcrumbList 类型（面包屑）
   - WebSite 类型（首页）

3. **canonical 标签**
   - 统一使用设置中的 siteUrl 作为基础
   - 分页页 canonical 指向第一页（或自引用）

4. **面包屑导航**
   - 位置：文章详情页顶部
   - 结构：首页 > 分类名 > 文章标题
   - 同时注入 JSON-LD 结构化数据

### 4.2 点赞/喜欢功能设计

**合规性说明**：纯计数功能，无用户发布内容，符合个人备案

**实现方案：**
- 前端：心形按钮 + 数字显示
- 后端：`like_num` 字段（文章表新增）
- 去重：IP + Cookie（24 小时内同一 IP 同一浏览器只计一次）
- 存储：Redis 计数 + 定时回写 MySQL（避免频繁写库）

### 4.3 Sitemap 增强设计

**当前问题**：只有基础 URL 列表

**增强内容：**
1. **首页**：priority=1.0, changefreq=daily
2. **分类/标签/专题/归档**：priority=0.6, changefreq=weekly
3. **文章**：
   - 置顶文章：priority=1.0
   - 阅读量 >1000：priority=0.9
   - 阅读量 >500：priority=0.85
   - 30 天内发布：priority=0.85
   - 其他：priority=0.8
4. **图片 sitemap**：有封面图的文章加入 image:image 标签
5. **lastmod**：使用文章更新时间

## 六、风险与注意事项

### 6.1 合规风险

| 风险点 | 等级 | 应对措施 |
|--------|------|----------|
| 评论功能 | 高 | 不做评论，保持纯展示 |
| 用户上传内容 | 高 | 仅管理员可发布 |
| 商业广告 | 中 | 不放商业广告，仅个人打赏 |
| 敏感内容 | 中 | 文章发布前自查 |

### 6.2 技术风险

| 风险点 | 等级 | 应对措施 |
|--------|------|----------|
| SEO 效果不明显 | 中 | 先做轻量方案，观察效果再决定是否 SSR |
| 点赞功能被刷 | 低 | IP + Cookie 双重限制，Redis 限流 |
