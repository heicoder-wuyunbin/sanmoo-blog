# 内容运营工作流

## 1. 概述

本文档定义 Sanmoo Blog 从选题到分发的完整内容运营流程，确保内容生产、组织、发布、分发的标准化和可追溯性。

## 2. 内容运营主流程

```
选题 → 写作 → 归类 → 专题编排 → 定时发布 → Web 展示 → 小程序分发 → 搜索收录 → 数据回看 → 二次复用
```

### 2.1 选题阶段

**目标**：确定内容方向，避免重复选题

**流程**：
1. 从个人技术实践、学习心得、项目经验中提取选题
2. 检查现有文章和专题，避免内容重复
3. 确定选题所属分类和可能关联的专题

**产出物**：
- 选题清单（可在后台草稿中记录）

### 2.2 写作阶段

**目标**：产出高质量原创内容

**流程**：
1. 在后台创建文章草稿（状态：`draft`）
2. 填写标题、摘要（description）、正文
3. 设置封面图（titleImage，可选）
4. 编写完成后进入审核阶段

**产出物**：
- 文章草稿

### 2.3 归类阶段

**目标**：将文章归入合适的分类和标签体系

**流程**：
1. 为文章选择主分类（category）
2. 为文章添加标签（tags），建议 3-5 个
3. 检查分类和标签是否已存在，优先复用已有标签

**产出物**：
- 分类关联
- 标签关联

### 2.4 专题编排阶段

**目标**：将相关文章组织成专题，形成知识体系

**流程**：
1. 确定文章是否属于某个专题
2. 如属于，将文章关联到对应专题（topic）
3. 调整专题内文章的排序，确保阅读顺序合理

**产出物**：
- 专题关联
- 专题内文章排序

### 2.5 发布阶段

**目标**：将文章发布到 Web 和小程序

**流程**：
1. 设置发布时间（可立即发布或定时发布）
2. 定时发布：设置 `scheduled_at` 字段，状态保持 `draft`，系统自动在指定时间发布
3. 立即发布：将状态改为 `published`
4. 发布后自动同步到 Web 和小程序

**产出物**：
- 已发布文章（状态：`published`）

### 2.6 分发阶段

**目标**：确保内容在 Web 和小程序双端正确展示

**流程**：
1. Web 端：文章出现在首页、分类页、标签页、专题页、归档页
2. 小程序端：文章出现在首页、分类页、搜索页
3. 检查两端展示一致性（标题、摘要、标签、分类）

**产出物**：
- Web 端文章展示
- 小程序端文章展示

### 2.7 搜索收录阶段

**目标**：确保文章可被搜索到

**流程**：
1. 如使用 MeiliSearch，文章发布后自动同步到搜索引擎
2. 检查搜索结果是否包含新文章
3. 如需手动同步，可在后台触发索引重建

**产出物**：
- 搜索索引更新

### 2.8 数据回看阶段

**目标**：跟踪文章表现，为后续选题提供依据

**流程**：
1. 在仪表盘查看文章阅读量排行
2. 查看分类/标签阅读量统计
3. 查看内容热度趋势
4. 分析高阅读量文章的共性

**产出物**：
- 数据洞察
- 选题优化建议

### 2.9 二次复用阶段

**目标**：将优质内容沉淀为专题，提升长期价值

**流程**：
1. 识别高阅读量、高收藏率的文章
2. 将相关文章组织成专题
3. 在首页"专题精选"板块推荐
4. 定期更新专题内容，保持时效性

**产出物**：
- 专题更新
- 内容复用

## 3. 内容状态流转

```
draft（草稿） → published（已发布） → archived（已归档）
```

- **draft**：文章正在编写或等待定时发布
- **published**：文章已发布，在 Web 和小程序可见
- **archived**：文章已归档，不再在首页展示，但仍可通过归档页访问

## 4. 内容组织层级

```
分类（Category）
  └── 文章（Article）
        ├── 标签（Tag）
        └── 专题（Topic）
```

- **分类**：文章的主分类，每篇文章只能属于一个分类
- **标签**：文章的关键词标签，每篇文章可有多个标签
- **专题**：相关文章的集合，形成知识体系

## 5. 内容发布规范

### 5.1 文章元信息

- **标题**：简洁明了，体现核心内容
- **摘要**：100-200 字，概括文章主要内容
- **封面图**：可选，建议尺寸 1200x630px
- **分类**：必须选择
- **标签**：建议 3-5 个，优先复用已有标签
- **专题**：如属于某个专题，必须关联

### 5.2 正文规范

- 使用 Markdown 格式
- 代码块需指定语言
- 图片需有 alt 文本
- 链接需可访问

### 5.3 发布时机

- **工作日**：上午 9-11 点，下午 2-4 点
- **周末**：上午 10-12 点
- **定时发布**：可提前设置，系统自动发布

## 6. 内容质量标准

### 6.1 原创性

- 所有内容必须为原创
- 如引用他人内容，需明确标注来源

### 6.2 准确性

- 技术内容需经过验证
- 代码示例需可运行

### 6.3 可读性

- 结构清晰，层次分明
- 语言简洁，避免冗余
- 适当使用图表、代码示例

### 6.4 价值性

- 解决实际问题
- 提供可操作的建议
- 有明确的收获点

## 7. 内容复盘机制

### 7.1 周复盘

- 查看本周发布文章数量
- 查看文章阅读量表现
- 识别高阅读量文章的共性

### 7.2 月复盘

- 查看本月内容发布趋势
- 分析分类/标签阅读量分布
- 调整下月选题方向

### 7.3 季度复盘

- 评估专题运营效果
- 优化内容组织策略
- 调整内容运营方向
