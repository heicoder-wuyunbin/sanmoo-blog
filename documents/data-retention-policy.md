# 数据保留与清理策略

维护任务每天凌晨 03:00 执行。清理失败会写入服务日志，下一次任务继续重试；管理员可通过维护接口手动触发。

| 数据 | 保留期 | 用途 |
| --- | ---: | --- |
| 访问日志 | 90 天 | 安全排障与基础访问统计 |
| 搜索历史 | 90 天 | 搜索质量优化 |
| 错误日志 | 180 天 | 故障排查 |
| 小程序阅读历史 | 180 天 | 读者阅读记录 |
| 文章日 PV 汇总 | 365 天 | 内容表现回看 |
| 推荐曝光记录 | 90 天 | 轻量推荐效果核对 |

冻结的用户画像、兴趣和标签表停止新增写入，不纳入自动运营分析。涉及用户删除请求时，应由删除接口清理关联的个人数据；历史数据清理前先完成备份与验证。
