# 专题运营规范

## 1. 概述

专题是 Sanmoo Blog 的核心知识沉淀能力。通过专题，将零散的文章组织成体系化的知识路径，帮助读者从入门到深入系统学习某个技术领域。

## 2. 专题定位

### 2.1 专题是什么

- 专题是围绕某个技术主题的系列文章集合
- 专题提供体系化的阅读路径，而非零散的文章堆砌
- 专题是个人知识沉淀的核心载体

### 2.2 专题不是什么

- 专题不是文章分类（分类是横向归类，专题是纵向深入）
- 专题不是标签（标签是关键词，专题是知识体系）
- 专题不是必须（不是所有文章都需要归入专题）

## 3. 专题创建规范

### 3.1 专题创建时机

- 当某个主题下已有 3 篇及以上相关文章时，可创建专题
- 当计划系统性地写某个主题系列文章时，可提前创建专题
- 当已有文章可以重新组织成知识体系时，可创建专题

### 3.2 专题命名规范

- 标题简洁明了，体现主题核心
- 避免过于宽泛（如"前端开发"）
- 避免过于狭窄（如"React 18 新特性"）
- 建议格式：`主题名` 或 `主题名 + 修饰语`
  - ✅ `Go 并发编程`
  - ✅ `微信小程序开发实战`
  - ❌ `前端`（太宽泛）
  - ❌ `React 18.2.0 useTransition 详解`（太狭窄）

### 3.3 专题描述规范

- 描述应说明专题的核心内容和目标读者
- 建议 50-150 字
- 格式建议：
  ```
  本专题围绕 [主题]，从 [起点] 到 [终点]，系统介绍 [核心内容]。
  适合 [目标读者] 阅读。
  ```

示例：
```
本专题围绕 Go 并发编程，从基础概念到高级实践，系统介绍 goroutine、channel、sync 包等核心内容。
适合有一定 Go 基础、希望深入理解并发编程的开发者阅读。
```

## 4. 专题内容编排

### 4.1 文章排序原则

专题内文章应按学习路径排序，而非按发布时间排序：

1. **入门篇**：基础概念、环境搭建、快速上手
2. **基础篇**：核心概念、基本用法、常见模式
3. **进阶篇**：高级特性、最佳实践、性能优化
4. **实战篇**：项目案例、踩坑经验、总结反思

### 4.2 文章关联规范

- 每篇文章只能属于一个专题（避免知识体系混乱）
- 如文章跨多个主题，选择最核心的专题关联
- 可在文章正文中通过链接引用其他专题的文章

### 4.3 专题更新规范

- 新增文章后，及时关联到对应专题
- 定期审视专题内文章的顺序，确保学习路径合理
- 如专题内容已过时，更新文章或标记为"历史内容"

## 5. 专题展示规范

### 5.1 Web 端展示

#### 首页专题精选

- 展示 3-4 个推荐专题
- 每个专题显示：标题、描述、文章数量
- 点击专题进入专题详情页

#### 专题列表页

- 展示所有专题
- 每个专题显示：标题、描述、文章数量、创建时间
- 支持按文章数量排序

#### 专题详情页

- 专题头部：标题、描述、文章数量、创建时间
- 学习路径提示：建议按顺序阅读
- 文章列表：按学习路径排序，每篇文章显示序号标记
- 每篇文章显示：标题、摘要、标签、阅读量、发布时间

### 5.2 小程序端展示

#### 首页专题入口

- 在首页展示专题入口
- 点击进入专题列表页

#### 专题详情页

- 专题头部：标题、描述、文章数量
- 文章列表：按学习路径排序
- 每篇文章显示：标题、摘要、阅读量、发布时间

## 6. 专题运营指标

### 6.1 核心指标

- **专题数量**：当前活跃专题总数
- **专题文章数**：每个专题包含的文章数量
- **专题阅读量**：专题内文章的总阅读量
- **专题完成率**：读者按顺序阅读专题文章的比例（如未来有数据支持）

### 6.2 运营目标

- 每个专题至少包含 3 篇文章
- 推荐专题的文章数量不少于 5 篇
- 定期更新专题内容，保持时效性

## 7. 专题推荐策略

### 7.1 首页推荐专题选择标准

1. **文章数量**：优先推荐文章数量较多的专题
2. **阅读量**：优先推荐总阅读量较高的专题
3. **时效性**：优先推荐近期有更新的专题
4. **完整性**：优先推荐知识体系较完整的专题

### 7.2 推荐专题数量

- 首页专题精选板块展示 3-4 个专题
- 如专题数量不足 3 个，不展示专题精选板块

## 8. 专题生命周期

### 8.1 专题创建

- 确定专题主题和范围
- 编写专题描述
- 关联已有文章（如有）

### 8.2 专题成长

- 持续产出相关文章
- 及时关联新文章到专题
- 调整文章排序，优化学习路径

### 8.3 专题成熟

- 专题文章数量达到 5 篇以上
- 专题阅读量稳定增长
- 知识体系相对完整

### 8.4 专题维护

- 定期更新过时内容
- 补充新的文章
- 如专题已不再维护，可在描述中说明
