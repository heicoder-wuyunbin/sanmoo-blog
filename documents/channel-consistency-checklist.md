# 渠道分发一致性清单

## 1. 概述

本文档定义 Web 端和小程序端内容分发的展示规范，确保两端在内容呈现上保持一致性，同时允许平台特性导致的合理差异。

## 2. 一致性原则

### 2.1 核心原则

- **内容一致**：同一篇文章在两端展示的标题、摘要、正文、标签、分类必须一致
- **语义一致**：两端的页面功能、信息层级、交互语义应保持一致
- **视觉差异**：允许因平台特性导致的视觉风格差异（Web 使用 Ant Design，小程序使用微信原生组件）

### 2.2 允许的差异

- **UI 组件**：Web 使用 Ant Design，小程序使用微信原生组件
- **交互方式**：Web 使用鼠标点击，小程序使用触摸操作
- **平台特性**：小程序可利用微信特有能力（如分享、书签、订阅消息）
- **响应式**：Web 需适配桌面和移动端，小程序主要适配移动端

## 6. 一致性维护流程

### 6.1 新增功能时

1. 确认功能是否在两端都需要实现
2. 如两端都需要，确保内容字段一致
3. 允许 UI 和交互方式的平台差异
4. 更新本文档的检查清单

### 6.2 修改现有功能时

1. 检查修改是否影响另一端
2. 如影响，同步修改另一端
3. 更新本文档的检查清单

### 6.3 定期审查

1. 每月审查一次两端一致性
2. 检查是否有新增的不一致项
3. 更新本文档的检查清单
