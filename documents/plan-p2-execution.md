# P2 阶段执行计划：强化内容运营主流程

## 目标

把内容生产到分发做成统一链路，提升内容复用效率。

## 现状评估

### 已完成的功能

1. **文章发布流程** ✅
   - 文章表结构完整（t_article, t_article_content）
   - 定时发布功能已实现（scheduled_at 字段）
   - 文章状态流转完整（draft/published/archived）
   - 后端 API 完整

2. **专题功能** ✅
   - 专题表结构完整（t_topic）
   - 文章-专题关联表完整（t_article_topic_rel）
   - 后端 API 完整
   - Web 和小程序都有专题页面

3. **分类和标签** ✅
   - 分类表（t_category）和标签表（t_tag）完整
   - 文章-分类、文章-标签关联完整
   - 后端 API 完整
   - Web 和小程序都有分类和标签页面

4. **归档功能** ✅
   - 后端归档 API 已实现（按月分组）
   - Web 有归档页面

5. **搜索功能** ✅
   - MeiliSearch 集成完整
   - 后端搜索 API 完整
   - Web 和小程序都有搜索页面

6. **推荐功能** ✅
   - 支持三种推荐策略（rule/weighted/cf）
   - 相关文章、热文、随机文章 API 完整

### 需要强化的功能

1. **专题运营规范** 🔶
   - 缺少专题运营规范文档
   - 专题在首页的展示不够突出
   - 专题详情页缺少"知识沉淀"的视觉引导

2. **内容分发一致性** 🔶
   - Web 和小程序的文章展示风格不统一
   - 专题展示在两端有差异
   - 搜索功能在两端体验不一致

3. **数据回看能力** 🔶
   - 缺少文章阅读数据统计
   - 缺少专题热度统计
   - 仪表盘缺少内容表现指标

## 执行计划

### 任务 1：强化专题作为知识沉淀核心

**目标**：将专题提升为一级运营能力，强化"知识沉淀"的视觉表达

**具体工作**：

1. **Web 端首页专题展示优化**
   - 在首页增加"专题精选"板块
   - 展示 3-4 个推荐专题
   - 每个专题显示文章数量和简介

2. **专题详情页优化**
   - 增加专题简介区域
   - 增加文章列表的"学习路径"提示
   - 增加专题统计（文章数、阅读量、收藏数）

3. **小程序端专题展示优化**
   - 首页增加专题入口
   - 专题详情页增加简介和统计

4. **后台专题管理优化**
   - 增加专题排序功能（推荐专题置顶）
   - 增加专题推荐标记

**涉及文件**：
- `sanmoo-vite/src/pages/web/Home.tsx`
- `sanmoo-vite/src/pages/web/TopicDetail.tsx`
- `sanmoo-mp/miniprogram/pages/index/index.ts`
- `sanmoo-mp/miniprogram/pages/topic-detail/index.ts`
- `sanmoo-vite/src/pages/admin/Topics.tsx`

### 任务 2：完善内容分发链路

**目标**：统一 Web 和小程序的内容呈现规则

**具体工作**：

1. **文章展示统一**
   - 统一文章列表的展示格式（标题、摘要、封面、标签、日期）
   - 统一文章详情页的展示格式（标题、元信息、正文、相关文章）
   - 统一代码块、图片、链接的渲染样式

2. **专题展示统一**
   - 统一专题列表的展示格式
   - 统一专题详情页的展示格式

3. **搜索功能统一**
   - 统一搜索结果的展示格式
   - 统一搜索高亮逻辑

4. **分类/标签展示统一**
   - 统一分类页的展示格式
   - 统一标签页的展示格式

**涉及文件**：
- `sanmoo-vite/src/pages/web/*.tsx`
- `sanmoo-mp/miniprogram/pages/**/*.ts`

### 任务 3：强化数据回看能力

**目标**：在仪表盘增加内容表现指标

**具体工作**：

1. **文章阅读统计**
   - 增加文章阅读量排行
   - 增加文章收藏量排行
   - 增加文章评论量排行（如果启用评论）

2. **专题热度统计**
   - 增加专题阅读量统计
   - 增加专题文章数统计

3. **内容发布趋势**
   - 增加按月发布文章数量趋势图
   - 增加按分类发布文章数量分布图

**涉及文件**：
- `sanmoo-server-go/internal/application/statistics/service.go`
- `sanmoo-server-go/internal/interfaces/http/handler/admin_dashboard_handler.go`
- `sanmoo-vite/src/pages/admin/Dashboard.tsx`

### 任务 4：输出内容运营规范文档

**目标**：制定内容运营流程图和专题运营规范

**具体工作**：

1. **内容运营流程图**
   - 绘制从选题到分发的完整流程图
   - 明确每个环节的职责和产出物

2. **专题运营规范**
   - 制定专题创建、编排、更新的规范
   - 制定专题推荐和排序的规则

3. **渠道分发一致性清单**
   - 列出 Web 和小程序的内容展示规范
   - 确保两端内容呈现一致

**产出物**：
- `documents/content-workflow.md`
- `documents/topic-operation-guide.md`
- `documents/channel-consistency-checklist.md`

## 执行顺序

1. **任务 1**：强化专题作为知识沉淀核心（Web 端）
2. **任务 2**：完善内容分发链路（Web + 小程序）
3. **任务 3**：强化数据回看能力（仪表盘）
4. **任务 4**：输出内容运营规范文档

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Web 和小程序 UI 框架差异大 | 中 | 制定统一的展示规范，但允许 UI 差异 |
| 数据统计需要新增表 | 低 | 优先使用现有 t_access_log 和 t_statistics 表 |
| 专题推荐需要新增字段 | 低 | 在 t_topic 表增加 sort_order 和 is_recommended 字段 |

## 时间估算

- 任务 1：2-3 小时
- 任务 2：3-4 小时
- 任务 3：2-3 小时
- 任务 4：1-2 小时

**总计**：约 8-12 小时
