## 三、产品语义统一方案

### 3.1 核心原则

1. **定位收敛**：所有文案必须体现"个人技术内容发布与知识整理平台"
2. **去平台化**：消除"用户管理""运营""社区""平台"等 B 端/C 端平台词汇
3. **去社交化**：不出现"关注""粉丝""好友""动态"等社交关系链词汇
4. **阅读导向**：小程序功能围绕"阅读、收藏、历史、订阅、轻量推荐"

### 3.2 需要改动的文案清单

| 位置 | 当前文案 | 目标文案 | 优先级 |
|------|----------|----------|--------|
| 后台菜单 | 用户管理 | 账号管理 | P0 |
| 后台菜单 | 微信用户 | 读者 | P0 |
| 后台菜单 | 角色管理 | 权限角色 | P1 |
| Web 前台 | 收藏 | 书签（或保持"收藏"） | P2 |
| Web 前台 | 浏览历史 | 阅读记录 | P2 |
| 小程序 mine | 我的 | 我的书斋 | P2 |
| 小程序 mine | 我的收藏 | 我的书签 | P2 |
| 小程序 mine | 浏览历史 | 阅读记录 | P2 |
| 全局导航标题 | Sanmoo Blog | 保持不变 | - |

### 3.3 需要弱化/隐藏的入口

| 入口 | 动作 | 理由 |
|------|------|------|
| 小程序 `logs` 目录 | 删除 | 未注册且废弃 |
| 后台 `Users` 子页面中的"用户注册"相关入口 | 隐藏（如存在） | 个人博客不需要公开注册 |
| Web 前台 `/favorites` 和 `/history` | 如果未登录则隐藏入口 | 需要登录态才展示 |

### 3.4 不需要改动的内容

以下内容完全符合个人备案定位，无需修改：
- 所有文章/分类/标签/专题相关页面和文案
- 搜索功能
- 关于/隐私政策页面
- 后台文章管理、仪表盘、文件管理、系统配置
- 小程序首页、分类、专题、标签、详情页
- 海报生成（属于文章分享功能）

将 README 从"通用博客系统"定位调整为"个人技术内容发布与知识整理平台"，消除平台化表达。

## 八、风险与后续建议

### 风险

| 风险 | 说明 | 建议 |
|------|------|------|
| 文案改动影响 SEO | 页面标题变更可能影响搜索引擎收录 | 后续实施时保留 meta title 不变，仅改页面内可见文案 |
| "我的"→"我的书斋"接受度 | 用户可能不习惯新名称 | 可先保留"我的"，后续 A/B 测试 |
| 后台菜单改名 | 管理员需要适应新菜单名 | 在 M1 实施时做平滑过渡 |

### 后续建议

1. **M1 阶段**：按本文档第三章执行具体文案修改
2. **M2 阶段**：按 README 改版提纲重写 README.md
3. **跨阶段注意**：本文档中标注"需要检查后端"的条目（如友链审核），需确认后端实际逻辑后再决定前端是否改动
4. **小程序 logs 目录**：确认无引用后可在 M1 阶段删除
